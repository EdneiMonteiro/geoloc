01

O Problema

Como verificar se um usuário está fisicamente presente no endereço cadastrado — em tempo real, usando apenas o celular?

Solução: GeoLoc

  • Usuário tira uma foto no app (iPhone)
  • O app captura as coordenadas GPS do dispositivo
  • O backend geocodifica o endereço cadastrado via Azure Maps
  • Calcula a distância entre as duas coordenadas
  • Retorna se o usuário está dentro de um raio de 50 metros

Casos de Uso

  • Validação de presença em visitas técnicas
  • Comprovação de entrega em endereço cadastrado
  • Confirmação de ponto de trabalho remoto
  • Verificação de endereço residencial para serviços financeiros

🗺️ Azure Maps — Geocodificação

O serviço Azure Maps Search Address API converte um endereço textual em coordenadas geográficas (latitude/longitude). Esta é a chamada central da solução.

📂 api/Services/AzureMapsService.cs — CHAMADA AO AZURE MAPS 
public async Task<GeoCoordinate?> GeocodeAddressAsync(string address)
{
    _logger.LogInformation("Geocoding address: {Address}", address);

    // Monta a URL da Azure Maps Search Address API
    var encodedAddress = Uri.EscapeDataString(address);
    var url = $"https://atlas.microsoft.com/search/address/json
              ?api-version=1.0
              &subscription-key={_subscriptionKey}
              &query={encodedAddress}
              &countrySet=BR&language=pt-BR&limit=1";

    // Faz a chamada HTTP GET ao Azure Maps
    var response = await _httpClient.GetAsync(url);
    response.EnsureSuccessStatusCode();

    // Parseia o JSON de resposta
    var json = await response.Content.ReadFromJsonAsync<JsonElement>();

    var results = json.GetProperty("results");
    if (results.GetArrayLength() == 0)
        return null;

    // Extrai as coordenadas do primeiro resultado
    var position = results[0].GetProperty("position");
    var lat = position.GetProperty("lat").GetDouble();
    var lng = position.GetProperty("lon").GetDouble();

    return new GeoCoordinate { Latitude = lat, Longitude = lng };
}

🔐 Autenticação do Azure Maps

O Azure Maps suporta dois métodos de autenticação. Esta PoC usa Shared Key por simplicidade. Para produção, recomenda-se Managed Identity.

🔑 Shared Key (esta PoC)

  • Chave estática (subscription-key)
  • Passada como parâmetro na URL
  • Simples de configurar
  • Rotação manual
REQUEST 
GET /search/address/json
    ?subscription-key={CHAVE}
    &query=Av. Paulista, 1000

🛡️ Managed Identity (produção)

  • Sem chaves — token OAuth gerenciado pelo Azure
  • Bearer token no header Authorization
  • Rotação automática
  • Auditável via Azure AD logs
REQUEST 
GET /search/address/json?query=Av. Paulista
x-ms-client-id: {CLIENT_ID}
Authorization: Bearer {TOKEN}

Setup da Managed Identity (5 passos)

# Passo Comando CLI
1 Habilitar Managed Identity na Function App az functionapp identity assign --name {func}
2 Atribuir role Azure Maps Data Reader az role assignment create --role "Azure Maps Data Reader"
3 Obter Client ID do Azure Maps az maps account show --query properties.uniqueId
4 Usar DefaultAzureCredential no código Substituir subscription-key por Bearer token
5 Desabilitar Shared Key (opcional) az maps account update --disable-local-auth true

Roles RBAC disponíveis

Role Permissão Uso
Azure Maps Data Reader Somente leitura (Search, Render, Route...) ✅ Recomendado para esta PoC
Azure Maps Search and Render Data Reader Apenas Search + Render Alternativa mais restritiva
Azure Maps Data Contributor Leitura + escrita + exclusão Apenas se criar/editar dados

📐 Validação de Raio — Fórmula de Haversine

A fórmula de Haversine calcula a distância entre dois pontos na superfície de uma esfera (a Terra), considerando sua curvatura. É o padrão para cálculos de distância geográfica com coordenadas GPS.

📂 api/Services/GeoCalculationService.cs — CÁLCULO DE DISTÂNCIA 
public class GeoCalculationService
{
    private const double EarthRadiusMeters = 6_371_000;  // Raio médio da Terra

    /// Calcula a distância em metros entre duas coordenadas geográficas
    /// usando a fórmula de Haversine.
    public double CalculateDistanceMeters(
        double lat1, double lon1,   // Coordenadas do dispositivo (GPS)
        double lat2, double lon2)   // Coordenadas do endereço (Azure Maps)
    {
        var dLat = DegreesToRadians(lat2 - lat1);
        var dLon = DegreesToRadians(lon2 - lon1);

        var a = Math.Sin(dLat / 2) * Math.Sin(dLat / 2) +
                Math.Cos(DegreesToRadians(lat1)) * Math.Cos(DegreesToRadians(lat2)) *
                Math.Sin(dLon / 2) * Math.Sin(dLon / 2);

        var c = 2 * Math.Atan2(Math.Sqrt(a), Math.Sqrt(1 - a));

        return EarthRadiusMeters * c;  // Distância em metros
    }
}

Como é usado na Function

📂 api/Functions/ValidateLocationFunction.cs — VALIDAÇÃO DO RAIO 
// 4. Calcula distância entre GPS do dispositivo e endereço geocodificado
var distanceMeters = _geoCalc.CalculateDistanceMeters(
    validationRequest.Latitude, validationRequest.Longitude,   // GPS do iPhone
    addressCoords.Latitude, addressCoords.Longitude);          // Azure Maps

// 5. Verifica se está dentro do raio de 50 metros
var isWithin = distanceMeters <= 50.0;  // RadiusMeters = 50
